Overview

Slick Modals is a HTML5 and CSS3 popup intended to be easily implemented into almost any website with some basic HTML and CSS knowledge. It is a plugin made for easy usage, customization and further development. It is not a CMS plugin, module or widget hence it needs to be installed manually into specific CMS. For further reading Slick Modal will be referenced as a plugin.

Supported browsers

  • Google Chrome 40+
  • Opera 40+
  • Firefox 30+
  • Internet Explorer 11+
  • Microsoft Edge

Plugin is tested in browsers listed above on Windows 10 platform. Older browsers (like Internet Explorer 10 and below) or browsers that don't support CSS3 and some Javascript features in full will not render plugin correctly. Plugin should work on almost all modern portable devices (iOS and Android) to but it was created and tested to work mainly on desktops and high-res tablets which support features used in the plugin.

Responsiveness

Although plugin has responsive settings, they are bound to the popup wrapper and will not affect the inner content. If you plan to use responsive content, it's best to adapt your content with CSS media queries. You can read more about media queries at W3Schools.

Plugin structure

Plugin is made of base HTML element which defines the popup content wrapper, CSS for basic design along with animations and jQuery plugin which controls the functionality of the popup.

HTML

Plugin HTML base is a wrapper element with ID or class and data-sm-init="true" attribute which is used as a target element when invoking and setting up the plugin options. For example:

    <div class="myPopup" data-sm-init="true">
        ... popup inner content ...
    </div>

data-sm-init="true" is mandatory as it prevents popup flicker on page load. Without it the plugin will not work.

Plugin dynamically creates all the necessary wrapping elements and attributes.

CSS

The CSS contains main layout definitions, buttons styles and animations:

  1. Reset and basics
  2. Popup positions
  3. Popup transitions
  4. Close buttons

jQuery plugin

jQuery plugin controls the functionality part of the popup. The invoke consists from definable popup settings and callbacks. Below is an example of a simple plugin invoke based on the HTML structure above:

    <script>
        jQuery('.myPopup').SlickModals({
            popup_position: 'center',
            popup_css: {
                'padding': '40px',
                'margin' : '30px'
            }
        });
    </script>

Plugin can also be chained with other jQuery methods. For example: jQuery('.myPopup').attr('title', 'foo').addClass('foo').SlickModals().

Usage

Before using the plugin please check the requirements:

  • Required: jQuery 1.8+ has to be loaded on your website
  • Optional: PHP 5.6+ in case you are using server side scripts of the plugin (for example: server time for date range restriction)

Upload plugin files

In order for the plugin to work it needs the CSS and Javascript files. Upload the following files from the /src/plugin/ folder to your web server:

  1. slickmodal.min.css
  2. jquery.slickmodal.min.js

Include the plugin files

Edit the <head> section of your page and include the plugin CSS. For example:

    <head>
        ... other head elements ...
        <link href="yourUploadPath/slickmodal.min.css" rel="stylesheet" type="text/css" />
        ... other head elements ...
    </head>

Include the plugin Javascript right before closing </body> tag on your page. For example:

    <body>
        ... other body elements ...
        <script src="yourUploadPath/jquery.slickmodal.min.js"></script>
    </body>

For performance it is always recommended to use minified version of both CSS and Javascript files (.min.).

Always load plugin Javascript file (jquery.slickmodal.min.js) after jQuery library has loaded on your site.

Be sure you have specified correct path which reflects the CSS and Javascript files location on your web server.

You can also include the plugin Javascript file inside <head> section if needed but after the jQuery library.

Create HTML structure

Start by creating a main popup wrapper which should have an id or a class attribute defined. The main wrapper will serve as a target element for the plugin invoke and a wrapper for the inner content. After creating the main wrapper insert the popup content inside.

It's best to place the popup wrapper after the plugin script call and right before closing </body> tag. For example:

    <body>
        ... other body elements ...
        <script src="yourUploadPath/jquery.slickmodal.min.js"></script>
        <div class="myPopup" data-sm-init="true">
            ... popup inner content ...
        </div>
    </body>

DO NOT nest the popup HTML inside other elements. It should always be a first child of the <body>.

You can put any HTML element inside the popup content; iframe, video, image or other elements.

You can also use a Quick Start template or other demo(s) from the package and copy the HTML for a reference and a starting point.

Plugin setup & settings

Create plugin invoke, place it below the popup main wrapper and adjust the settings to your likings. For example:

    <body>
        ... other body elements ...
        <script src="yourUploadPath/jquery.slickmodal.min.js"></script>
        <div class="myPopup" data-sm-init="true">
            ... popup inner content ...
        </div>
        <script>
            jQuery('.myPopup').SlickModals({
                ... plugin settings ...
            });
        </script>
    </body>

If you don't define the popupType within the settings the plugin will use the default value which is set to "none". With popupType not defined the popup can be opened only by setting the popup_reopenClass and adding the defined reopen class to your element or via plugin api. If you wish to use delayed, scrolled or exit popup you must define the type and additional settings for the chosen type.

Use the same selector as used within the plugin main wrapper in the step 3.

Options definitions

Below is a list of options with definitions, their types, usage examples or possible values.

restrict_hideOnUrlsarray

Defines the pages where popup should stay hidden. It can accept a string with path or URL and a regular expression via new RegExp object for pattern precise matching.

Example: ['/yourUrl/', new RegExp(/^(your)|(URL)/)]

restrict_cookieSetboolean

Enables or disables setting a cookie which is used to hide the popup after the cookie has been set.

Possible values: true | false

restrict_cookieNamestring

Defines the cookie name for the current popup instance. Each popup should have a unique cookie name. Popups which have the same cookie name will all be hidden.

Example: 'slickModalCookie-1'

restrict_cookieScopestring

Sets the cookie scope to entire domain or just the current page.

Possible values: 'domain' | 'page'

restrict_cookieDaysnumber

Sets the expiration days for the cookie. If 0 is set the cookie will expire after the current session is closed eg. visitor closes the active tab.

Possible values: 0 - 365

restrict_cookieSetClassstring

Defines a class of the element on the page which sets the cookie for specific popup instance. Useful if you wish to set the cookie with another element outside the popup.

Example: 'reopenSlick-1'

restrict_dateRangeboolean

Enables or disables date range restriction.

Possible values: true | false

restrict_dateRangeStartstring

Specifies the starting date for date range. It must use the following format: Y-M-D, H:M:S.

Example: '2018-10-24, 15:00:00'

restrict_dateRangeEndstring

Specifies the ending date for date range. It must use the following format: Y-M-D, H:M:S.

Example: '2019-03-22, 10:30:00'

restrict_dateRangeServerTimeboolean

Enables or disables usage of server time collected via Ajax.

Possible values: true | false

restrict_dateRangeServerTimeFilestring

File location for the server time PHP script, provided with the package inside src/plugin directory. You can also use your own script to retrieve the server time.

Example: 'folder/getTime.php'

restrict_dateRangeServerTimeZonestring

Defines the time zone, used for the server time. Reference list for available PHP time zones: PHP timezones list

Example: 'Europe/London'

restrict_showAfterVisitsnumber

Restricts the popup visibility for defined number of page loads. Value 1 will show the popup always. Any number above will start to count the visits.

Possible values: 1 - ∞

restrict_showAfterVisitsResetWhenShownboolean

Defines if the popup should be hidden again on the next page load after it has been shown to the visitor when restrict_showAfterVisits option is enabled.

Possible values: true | false

popup_typestring

Defines the popup type on page load.

Possible values: 'delayed' | 'scrolled' | 'exit' | 'instant' | 'none'

popup_delayedTime'string'

Sets the popup delayed time in seconds when popup_type: 'delayed' is used.

Example: '3s'

popup_scrollDistancestring

Sets the popup scroll distance when popup_type: 'scrolled' is used.

Example: '440px'

popup_scrollHideOnUpboolean

Defines is popup should stay hidden when visitors scroll back up upon using popup_type: 'scrolled'.

Possible values: true | false

popup_exitShowAlwaysboolean

Defines if exit popup should appear always when using popup_type: 'exit'.

Possible values: true | false

popup_autoCloseboolean

Sets the popup auto close after it has been opened.

Possible values: true | false

popup_autoCloseAfterstring

Defines the time in seconds when the popup will auto close upon opening.

Example: '5s'

popup_openWithHashstring

Enables opening the popup via URL hash. If empty it will disable this option.

Example: '#openMyPopup'

popup_redirectOnCloseboolean

Defines if visitor gets redirected to another URL after popup gets closed.

Possible values: true | false

popup_redirectOnCloseUrlstring

Redirect URL after popup gets closed.

Example: 'https://example.com'

popup_redirectOnCloseTargetstring

Sets the target of the redirection when using popup_redirectOnClose feature.

Possible values: '_blank' | '_self'

popup_redirectOnCloseTriggersstring

Defines which elements should trigger the redirection upon closing the popup. Possible values can be combined together.

Possible values: 'overlay' | 'button' | 'close' | 'overlay button' | 'overlay close' | 'button close' | 'overlay button close'

popup_positionstring

Defines the popup position on desktop devices.

Possible values: 'topLeft' | 'topCenter' | 'topRight' | 'bottomLeft' | 'bottomCenter' | 'bottomRight' | 'left' | 'right' | 'center'

popup_animationstring

Defines the popup animation upon opening. Closing animation will apply automatically for most of the animations. Animations which don't support closing animation use fade out effect.

popup_closeButtonEnableboolean

Defines the closing button style.

popup_closeButtonAlignstring

Public API

Public methods

Plugin has some useful public methods which can be used to modify the popup behaviour or looks. In case you need to alter the plugin core to fit your needs a non minified version is available which can be used to customize or adjust existing functionality of the plugin. Below are definitions for each method and some common usage examples. All methods are passed as a parameter and some are accompanied with additional parameters.

openPopup()function

Opens the existing popup instance.

Usage: 'openPopup'

    jQuery('.foo').SlickModals('openPopup');

closePopup()function

Closes the existing popup instance.

Usage: 'closePopup'

    jQuery('.foo').SlickModals('closePopup');

styleElement()function

Sets CSS attributes for the popup or overlay for the current instance.

Usage: 'styleElement', 'popup | overlay', object

    jQuery('.foo').SlickModals('styleElement', 'popup', {
        'border-radius': '20px',
        'background': '#fefefe'
    });
    jQuery('.foo').SlickModals('styleElement', 'overlay', {
        'background': 'rgba(255,255,255,.7)',
        'animation-delay': '1s'
    });

popupPosition()function

Sets the position for the current popup instance.

Usage: 'popupPosition', location

    jQuery('.foo').SlickModals('popupPosition', 'right');

setEffect()function

Sets the effect for the popup or overlay for the current instance.

Usage: 'setEffect', 'popup | overlay', effect

    jQuery('.foo').SlickModals('setEffect', 'popup', 'zoomIn');
    jQuery('.foo').SlickModals('setEffect', 'overlay', 'slideLeft');

setType()function

Sets the type and value (depending on the type - exit and instant require no value) for the current popup instance.

Usage: 'setType', type, val

    jQuery('.foo').SlickModals('setType', 'delayed', '3s');
    jQuery('.foo').SlickModals('setType', 'scrolled', '400px');
    jQuery('.foo').SlickModals('setType', 'exit');
    jQuery('.foo').SlickModals('setType', 'instant');

autoClose()function

Sets auto close for the current popup instance. Value is needed only when enabling auto close.

Usage: 'autoClose', 'enable | disable', val

    jQuery('.foo').SlickModals('autoClose', 'enable', '2s');
    jQuery('.foo').SlickModals('autoClose', 'disable');

destroy()function

Destroys the current popup instance.

Usage: 'destroy'

    jQuery('.foo').SlickModals('destroy');

Methods chaining

Public methods can also be easily chained. Below is an example:

    jQuery('.foo').SlickModals('popupPosition', 'left')
                  .SlickModals('setEffect', 'popup', 'fadeIn')
                  .SlickModals('setType', 'delayed', '2s')
                  .SlickModals('openPopup');

Frequently asked questions

How to close the popup on element click?

You can add an attribute data-sm-close="true" to any element inside the popup and it will trigger closing that popup on click. For example:

    <div class="myPopup" data-sm-init="true">
        ... popup inner content ...
        <button data-sm-close="true">Close the popup</button>
    </div>

The same element can also set a cookie to hide the popup. Simply enable cookie settings and define restrict_cookieSetClass. Then add that class to the element that closes the popup. For example:

    <div class="myPopup" data-sm-init="true">
        ... popup inner content ...
        <button class="setMyPopupCookie" data-sm-close="true">Close the popup</button>
    </div>
    <script>
        jQuery('.myPopup').SlickModals({
            restrict_cookieSet: true,
            restrict_cookieName: 'hideMyPopup',
            restrict_cookieDays: '14',
            restrict_cookieSetClass: 'setMyPopupCookie'
            ... other settings ...
        });
    </script>

How do I setup multiple popups on the same page?

In order to have multiple popups on the same page you should create a HTML wrapper for each instance. If the options of the popups will be the same then you can use one invoke for all of them.

Example 1: Three popups with the same options

    <div class="myPopup" data-sm-init="true">
        ... popup 1 inner content ...
    </div>
    <div class="myPopup" data-sm-init="true">
        ... popup 2 inner content ...
    </div>
    <div class="myOtherPopup" data-sm-init="true">
        ... popup 2 inner content ...
    </div>
    <script>
        jQuery('.myPopup, .myOtherPopup').SlickModals({
            popup_position: 'bottomRight',
            popup_css: {
                'padding': '40px',
                'margin' : '30px',
                'padding' : '30px',
                'background' : '#333'
            }
            ... other settings ...
        });
    </script>

Example 2: Two popups with different options

    <div class="myPopup-1" data-sm-init="true">
        ... popup 1 inner content ...
    </div>
    <div class="myPopup-2" data-sm-init="true">
        ... popup 2 inner content ...
    </div>
    <script>
        jQuery('.myPopup-1').SlickModals({
            restrict_cookieSet: true,
            restrict_cookieName: 'myPopup-1',
            restrict_cookieDays: '14',
            popup_type: 'delayed',
            popup_delayedTime: '2s',
            popup_closeButtonStyle: 'text simple',
            popup_position: 'bottomRight',
            popup_css: {
                'padding': '40px',
                'margin': '30px',
                'background': '#fefefe'
            }
            ... other settings ...
        });
        jQuery('.myPopup-2').SlickModals({
            overlay_isVisible: false,
            content_animate: true,
            content_animation: 'bounceInLeft',
            popup_position: 'center',
            popup_css: {
                'padding': '20px',
                'background': '#fff'
            },
            mobile_show: false
            ... other settings ...
        });
    </script>

How do I make popup responsive?

Plugin already has an option to define mobile rules via mobile_css option. But those options control the CSS only for popup the wrapper hence you need to create mobile or responsive rules for the inner content manually with CSS media queries. More info about media queries can be found at W3Schools.

How to install plugin into CMS?

Popup is not made as a plugin, module or extension for any CMS hence it needs to be installed manually. In order to install it into a CMS you can follow the Usage guide. Before starting you should be familiar with editing the appropriate template files and if needed create backups.

How to use Settings Generator

Popup settings generator is made mainly as a quick settings generator but it also has a simple preview so you can see the popup wrapper styling in real time. Simply select the settings you need, adjust the CSS styling and copy the generated code to your HTML. Settings Generator is currently in Beta so bugs might appear.

How can i make the overlay to be a background image?

You can set a background image simply by editing overlay's settings upon invoke. For example:

    <script>
        jQuery('.myPopup').SlickModals({
            ... other settings ...
            overlay_css: {
                'background': 'url('path/image.jpg') 0 0 no-repeat'
            }
        });
    </script>

Are the demo(s) fully functional?

Demos, provided within the plugin package are not fully functional (for example: contact form doesn't send emails, Mini cart doesn't update actual shopping cart etc...) as they're main purpose is only to demonstrate some of the usage possibilities and popup capabilities. All the styling and the functionality within the popup demos content and it's element(s) is made only as a preview to help get you started using the popup more quickly or get an idea for what can popup be used. Scripts in the popup demos are created for a single popup instance on the page. Support for the demos is not provided. Any additional styling, customizations and functional modifications/upgrades are left for you to implement.

How to upgrade from older version of the plugin?

In order to upgrade the plugin please check the changelog.txt which contains notes regarding the upgrade.

Tips & Tricks

Use minified version

For better performance use minified versions of the plugin core files - both CSS and Javascript. They are located in the src/plugin directory.

Avoid mixing same popup types

Although the popup has a safety feature built in which prevents multiple popups showing at the same time you should avoid using the same popup types at once / on the same page.

Avoid overdoing blur or scale effect

Blur & scale effects are appealing but they can also affect the site performance as the browser has to redraw all the elements upon firing the animation. This can lead to higher CPU or GPU usage, throttling and slower animation transitions. It's recommended to use lower values for blur and values closer to 1 for scale effect.

Use browser debug tools

If there is an issue with the plugin you can also open debug / developer tools inside the browser and see if there is an error or a message from the plugin within the console pane. This can often point out quick solution for the issue.

Troubleshooting

Uncaught TypeError: $(...).SlickModals is not a function

If you see this error in the console pane inside the browser developer tools it could means either:

  1. that the plugin is not loaded after jQuery library or
  2. jQuery library is missing on the website or
  3. there is a conflict with $ namespace and you should rename all $ symbols with jQuery within the plugin invoke.

Popup has no styling

If the popup displays only bare HTML without any styling it is missing the CSS call inside <head> section. In order to fix this please upload the plugin CSS (slickmodal.min.css) file to your web server and insert it into the <head> section. See Usage section for more information.

Page animations are not working on page load

This can happen due to the nature of how the browser loads the web page. This means that the page effect function within the plugin is searching for the body elements but they currently don't exist as the browser hasn't finished loading all the elements on the page. To solve this you should wrap the plugin invoke into a onload function which will wait for all the page elements to render and afterwards the page effect will work normally. For example:

    <script>
        jQuery(window).on('load', function () {
            jQuery('.myPopup').SlickModals({
                ... popup settings ...
            });
        });
    </script>

Why does my popup look messy?

If the demo (popup inner content) doesn't look as within the package preview it's due to CSS overriding from your website template. In that case it's best to inspect which CSS rules are overriding the popup styles and adjust them. Also if you use different font family on your page the fonts my distort the layout and design.

I have installed the popup correctly but it still doesn't show up

If you don't define the popupType within the settings the plugin will use the default value which is set to "none". With popupType not defined the popup can be opened only by setting the popup_reopenClass and adding the defined reopen class to your element or via plugin api. So for example if you wish the popup to show up after 5 seconds you should define the popupType like this:

    <script>
        jQuery('.myPopup').SlickModals({
            popup_type: 'delayed',
            popup_delayedTime: '5s',
            ... other plugin settings ...
        });
    </script>
